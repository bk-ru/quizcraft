Поэтапная генерация квиза вместо одной длинной страницы

Пример показывает, как можно разделить текущий сценарий на отдельные экраны: пользователь видит только текущий этап, понимает следующий шаг и может вернуться назад без потери контекста.

Идея будущей основной разработки Сохранить текущие возможности, но разнести их по шагам: загрузка документа, параметры, генерация, результат, редактирование и экспорт.

1. Загрузите документ

На первом экране оставляем только выбор файла и понятную проверку: формат, размер, извлекаемый русский текст.

Первый шаг

Что показываем после загрузки

  • Имя файла и размер.
  • Статус извлечения текста.
  • Предупреждение для PDF без текстового слоя.
  • Кнопку перехода к параметрам.

2. Настройте квиз

Второй экран собирает только параметры генерации. Пользователь не отвлекается на редактор, экспорт и технические детали результата.

Настройки

3. Отслеживайте обработку

Во время генерации показываем прогресс, таймер, возможность отмены и понятные сообщения, где именно возникла проблема.

Поведение ошибок

  • Если backend недоступен, показываем действие: запустить .\run-backend.ps1.
  • Если LM Studio недоступен, показываем действие: открыть LM Studio, загрузить модель и включить server mode.
  • Для RAG отдельно показываем статус кеша и повторного использования embeddings.

4. Проверьте результат

После генерации пользователь попадает на отдельный экран просмотра. Здесь видны вопросы, ответы, режим генерации и кнопка перехода к редактированию.
